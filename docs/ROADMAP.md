# Roadmap

This roadmap describes the current engineering direction for KAMIYO Protocol.

## Near Term

- Improve reliability and integration testing for service runtimes
- Expand trust-layer replay and outbox recovery tooling
- Harden deployment and observability standards across services
- Continue formal verification coverage expansion for critical invariants

## Mid Term

- Improve cross-chain interoperability and settlement flows
- Expand SDK ergonomics for agent frameworks
- Standardize production runbooks for service operations
- Reduce operational overhead for oracle and dispute workflows

## Long Term

- Stronger trust guarantees for autonomous agent-to-agent commerce
- Broader formal verification and property-based testing coverage
- Higher-throughput, lower-latency execution paths for trust-critical flows

## Non-Goals

- Backwards compatibility at all costs when correctness or safety requires protocol evolution
- Shipping features without corresponding observability and operational safeguards
