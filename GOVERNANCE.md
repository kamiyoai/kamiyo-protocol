# Governance

## Maintainers

Repository maintainers are responsible for:

- triaging issues and pull requests
- setting release direction and priorities
- enforcing contribution and security policies

## Decision Process

- Day-to-day changes are handled through normal pull request review.
- Cross-cutting or breaking changes require maintainer sign-off.
- Security-sensitive changes may require private coordination before merge.

## Release Discipline

- Changes to protocol-critical components should include explicit test evidence.
- Breaking changes must include migration notes in the pull request.
- Major operational changes should include rollback guidance.

## Communication

Use GitHub Issues and pull requests for project discussions and decisions.
